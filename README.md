# CTA Report Automation — Batch XML → PDF with Excitation Graph Rendering

> **Turned a 4+ hour daily manual task into a 5-minute automated pipeline** at Shubhada Polymers Products Pvt. Ltd., a manufacturing company in Nashik.

---

## 🧩 The Problem

Current Transformer (CT) test reports are generated by **CT Analyser (CTA) software** as XML files. The manager's daily workflow:

1. Open each XML in a browser
2. Print/export as PDF to send to customers
3. **Result:** Every single PDF was missing the Excitation Graph

The browser threw this error where the graph should be:
```
Browser does not support graphics object!
```

With **1000+ XML reports per batch**, this manual workaround was consuming **4+ hours every single day**.

---

## ✅ The Solution

A Python automation script that:

1. **Batch-processes 1000+ XML files** in parallel (configurable worker threads)
2. Applies the embedded **XSLT stylesheet** to transform XML → HTML (replicating the CTA report layout exactly)
3. **Reverse-engineers and replicates the CTA Excitation Graph** using `matplotlib` — log-log scale, IEC/ANSI knee-point dashed lines, dynamic axis ranges, SI unit tick labels (µA, mA, A), vendor-style formatting
4. **Injects the rendered graph** as a base64-encoded PNG directly into the HTML, replacing the browser's unsupported placeholder
5. Converts the final HTML → PDF using **headless Chrome** or **wkhtmltopdf**
6. **Merges all individual PDFs** into one master PDF
7. Cleans up all intermediate files automatically

**Result: 4+ hours → under 5 minutes.**

## ⚙️ How It Works

```
XML Files (1000+)
      │
      ▼
  XSLT Transform ──────────────────────────────────────┐
      │                                                 │
      ▼                                                 ▼
  HTML Report                             Extract V/I data from XML
      │                                   (UCTrms / ICTrms MeasPoints)
      │                                                 │
      │                                                 ▼
      │                                   Build Excitation Plot (matplotlib)
      │                                   - Log-log axes
      │                                   - IEC/ANSI knee-point dashed lines
      │                                   - Dynamic axis ranges + SI units
      │                                                 │
      ▼                                                 │
  Inject Plot PNG (base64) ◄───────────────────────────┘
  (replaces "Browser does not support graphics object!")
      │
      ▼
  HTML → PDF (headless Chrome / wkhtmltopdf)
      │
      ▼
  Merge all PDFs → Master PDF
      │
      ▼
  Cleanup intermediate files
```

## 📊 Excitation Graph Details

The graph faithfully replicates the CTA vendor output:

- **Log-log scale** (both axes)
- **Dynamic axis limits** snapped to 1-2-5 engineering decades
- **Knee-point reference lines** — horizontal (Vk) and vertical (Ik) dashed lines, sourced directly from the XML's `KneePoints` node per IEC 61869-2 / IEC-1 / IEC-6 / ANSI-30 / ANSI-45 standards
- **SI unit tick labels**: µA (micro), mA (milli), A — auto-selected based on data range
- **Configurable tick density**: minimal / normal / dense
- **Thread-safe rendering** via `threading.Lock` for parallel batch use
- Supports voltage fields: `UCTrms`, `UCorerms`, `UCTrect`

## 🚀 Usage

### Prerequisites
```bash
pip install lxml PyPDF2 matplotlib numpy
# Plus: Google Chrome/Chromium OR wkhtmltopdf
```

### Basic Run
```bash
python cta_report_automation_with_plot.py \
  --xml-dir ./reports/xml \
  --out-dir ./reports/pdf \
  --master-pdf ./reports/master_output.pdf
```

### With All Options
```bash
python cta_report_automation_with_plot.py \
  --xml-dir ./xml_reports \
  --out-dir ./pdf_output \
  --master-pdf ./CTA_Master.pdf \
  --engine chrome \
  --workers 8 \
  --excitation-v UCTrms \
  --kneepoint-standard IEC_69_2 \
  --x-ticks minimal \
  --axis-labels vendor \
  --force
```

### CLI Reference

| Argument | Default | Description |
|---|---|---|
| `--xml-dir` | *(required)* | Folder containing XML report files |
| `--out-dir` | *(required)* | Output folder for individual PDFs |
| `--master-pdf` | *(required)* | Path for the merged master PDF |
| `--xsl` | auto-detect | Override XSL stylesheet |
| `--engine` | `chrome` | PDF engine: `chrome` or `wkhtmltopdf` |
| `--workers` | `4` | Parallel processing threads |
| `--excitation-v` | `UCTrms` | Voltage field: `UCTrms`, `UCorerms`, `UCTrect` |
| `--kneepoint-standard` | `IEC_69_2` | Standard for knee-point: `IEC_69_2`, `IEC_1`, `IEC_6`, `ANSI_30`, `ANSI_45` |
| `--force-kneepoint` | — | Override knee-point manually: `Vk,Ik` e.g. `3.797,0.04609` |
| `--x-ticks` | `minimal` | Tick density: `minimal`, `normal`, `dense` |
| `--axis-labels` | `vendor` | `vendor` (I/V, V/V) or `classic` (I [A], U [V]) |
| `--force` | off | Reprocess XMLs even if PDF exists |
| `--no-excitation-plot` | off | Skip graph injection |
| `--debug-dump-html` | off | Save intermediate HTML for debugging |
| `--debug-first-only` | off | Process only first XML (quick test) |
